# 专注守卫

Windows 本地专注助手。源码和安装包发布于：

https://github.com/975952/FocusGuard-Windows

一个完全在本机运行的 Windows 专注助手。开始专注后，它会检查：

- 你是否长时间没有操作键盘和鼠标；
- 当前应用是否在允许列表中；
- 当前窗口标题是否包含允许的关键词；
- 跑偏是否超过你设置的宽限时间。

触发规则后，专注守卫会弹出置顶提醒。提醒语气可以选“温和”“直接”或“暴躁”。

## 启动

普通用户可前往 GitHub 的 **Releases** 页面下载 ZIP，完整解压后再启动。

双击 `启动专注守卫.vbs`。它会安静地在后台启动，不显示命令窗口。

如果电脑禁用了 VBS，请双击 `启动专注守卫.cmd`，功能相同，但会保留一个命令窗口。

最小化后应用会保留在任务栏并继续运行；点击右上角关闭、任务栏“关闭窗口”或按 `Alt+F4` 都会完全退出。需要后台继续监测时，请使用主界面的“最小化”按钮。

可在“应用选项”中开启“随 Windows 启动”。它只写入当前用户启动项，不需要管理员权限；下次登录后会以最小化状态运行，不会主动弹出旧复盘，直到你打开主窗口。

`FocusGuard.ps1`、`FocusGuard.Core.ps1`、`FocusGuard.Data.ps1`、`FocusGuard.Session.ps1`、`FocusGuard.App.ps1`、`FocusGuard.Main.xaml`、`FocusGuard.Reminder.xaml`、`FocusGuard.Summary.xaml`、`FocusGuard.History.xaml` 和 `FocusGuard.Styles.xaml` 需要放在同一文件夹中。

## 最好用的设置方式

1. 写下本轮唯一要完成的任务。
2. 在“允许范围”里逐项添加工作软件进程名，例如 `code`、`winword`、`excel`；支持 Enter 添加、Delete 或按钮批量移除。
3. 浏览器尽量逐项添加网页标题关键词，例如项目名、课程名或文档名。
4. 不建议直接允许 `chrome` 或 `msedge`，因为这样会连它们打开的娱乐网页一起放行。
5. 第一次建议使用 45 分钟、无操作 180 秒、跑偏宽限 25 秒。

界面提供 25、45、60 分钟快捷设置。提醒出现时可按 `Enter` 立即返回任务，或按 `Esc` 休息 5 分钟；休息期间专注倒计时会冻结。

## 专注评分与分析

每轮专注结束（包括提前结束）后，应用会自动给出 0–100 分，并展示详细分析：

- 专注质量（50 分）：有效监测时间中，处于允许范围的比例；
- 完成进度（30 分）：实际完成的计划时长；
- 跑偏恢复（10 分）：提醒后重新回到专注范围的比例；
- 节奏连续（10 分）：暂停、提醒后休息及暂停占比。

结果页不会机械地把每项都讲一遍。它先从当轮数据中挑出最关键的一件事，再给出证据和一个下一步动作；完整数据说明仍可展开核对。只有四个评分依据全部满分时才会得到 100 分；任何非满分结果都会额外出现一段直白的 “tell off”，点出最大的丢分项和必须改的动作，不进行人身攻击。

非满分结果不能直接略过：必须滚动到复盘确认区，勾选已经看清最大缺口和下一轮动作后，关闭按钮才会解锁；窗口右上角关闭同样会被拦截。确认时间会写入本机历史。如果强行退出导致复盘未完成，下次启动会重新弹出这条结果。

主界面的“历史汇总”会打开专注成绩档案，包括：

- 总轮次、平均得分、按时完成率和累计有效进行时长；
- 最近 10 轮分数走势、近期与个人基线的变化，以及样本是否足以判断；
- 最近 18 周“质量投入轨迹”：颜色同时考虑有效分钟与得分，不用单纯挂机时长制造深色；
- 连续性、完成率和反复缺口组成的长期分析；数据不足时明确保留判断，不强行归类；
- 最近 100 轮明细，以及“满分”“已复盘”“待复盘”状态；
- 双击任意一轮，重新查看当时的完整评分、分析、建议和 tell off。

评分历史最多保留最近 100 轮，仅保存在本机。

主窗口和成绩档案都支持像编辑器一样即时拖动中间细把手，并会记住窗口大小与面板宽度；双击把手可恢复推荐宽度，“应用选项”中可以一键恢复整套推荐布局。明细列会随窗口宽度调整，空间不够时提供横向滚动，而不会裁掉右侧文字。

## 隐私

专注守卫不截屏、不读取按键内容、不记录浏览历史，也不上传任何数据。它只在内存中读取 Windows 提供的空闲时长、当前应用进程名和窗口标题。设置保存在：

`%APPDATA%\FocusGuardCN\settings.json`

删除这个文件即可恢复默认设置。

若监测遇到异常，去重后的错误信息会写入：

`%APPDATA%\FocusGuardCN\focusguard.log`

软件不进行语音播报。用户可以选择是否播放 Windows 系统提示音；关闭开关后完全静音。具体文案会自动组合并轮换，但始终保持用户选择的语气，不会在“温和”“直接”“暴躁”之间自动切换；最近 48 小时使用过的文案不会重复。历史仅保存在本机：

`%APPDATA%\FocusGuardCN\reminder-history.json`

每轮评分和分析保存在：

`%APPDATA%\FocusGuardCN\session-history.json`

## 已知边界

- 它依据应用名和窗口标题判断，无法理解屏幕内容。
- 某些应用不会提供有意义的窗口标题，需要按进程名放行。
- 阅读纸质材料或长时间看屏幕时可能被判定为“无操作”，可把无操作阈值调高。
- 这是行为提醒工具，不是无法绕过的系统封锁软件；你始终可以暂停或退出。

## 开发与构建

- 自检（语法 + 核心逻辑冒烟）：`powershell -NoProfile -STA -File FocusGuard.ps1 -SelfTest`
- 单元测试（需要 Pester 5.x+）：`Invoke-Pester -Path .\tests`
- 打包发布：运行 `build.ps1`，依次完成自检与单元测试，并在 `dist` 中生成 `FocusGuard-Windows-v<版本>.zip`。版本号取自 `FocusGuard.Core.ps1` 顶部的 `$script:FocusGuardVersion`，同时会显示在主窗口标题栏。
- 每次推送由 GitHub Actions 运行自检、PSScriptAnalyzer（错误级别）和单元测试，配置见 `.github/workflows/ci.yml`。
